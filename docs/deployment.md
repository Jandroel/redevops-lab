# Deployment

ReDevOps Lab is not deployed yet. Phase 7 prepares the project structure, environment variables, and notes for a future split deployment.

## Frontend - Vercel

- Project root: `apps/web`
- Install command: `pnpm install`
- Build command: `pnpm build`
- Local development command: `pnpm dev:web`
- Required environment variable: `NEXT_PUBLIC_API_URL`

Example:

```txt
NEXT_PUBLIC_API_URL=https://redevops-lab-api.example.com/api
```

The frontend must not contain provider API keys. It only calls the public API URL.

## Backend - Railway

- Project root: `apps/api`
- Install command: `pnpm install`
- Build command: `pnpm build`
- Start command: `pnpm start`
- Required runtime variables: `PORT`, `CORS_ORIGIN`
- Optional runtime variables: `GITHUB_TOKEN`, AI provider variables

Example:

```txt
PORT=3001
CORS_ORIGIN=https://redevops-lab.example.com
GITHUB_TOKEN=
AI_ENABLED=false
AI_PROVIDER=mock
AI_MODEL=
AI_API_KEY=
AI_BASE_URL=
AI_MENTOR_MODE=learning
AI_TEMPERATURE=0.3
AI_TIMEOUT_MS=20000
```

Set `CORS_ORIGIN` to the deployed Vercel URL when deploying the API. Keep `AI_API_KEY` backend-only and leave AI disabled unless a provider is intentionally configured.

## API Surface

```txt
GET  /api/health
POST /api/analyze
GET  /api/reports/demo
GET  /api/reports/:id
GET  /api/reports/:id/export
GET  /api/docs
```

## Current Boundaries

- No database is connected yet.
- No login, saved history, billing, private repo support, or autonomous agents.
- Reports generated by `/report/[id]` still rely on session storage unless the API returns a demo report.
- AI mentor is optional, mock mode is the default, and deterministic report facts remain the source of truth.

## Database - Future

PostgreSQL is not used in Phase 7. A local Docker Compose service is included so Prisma and persistence can be introduced later without changing the repository shape.

Planned local URL:

```txt
postgresql://redevops:redevops@localhost:5432/redevops_lab
```
