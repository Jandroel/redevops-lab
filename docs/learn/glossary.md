# Glossary

## CI

Continuous Integration. Automated validation for code changes before they are merged.

## CD

Continuous Delivery or Deployment. The process of preparing or shipping a version after checks pass.

## Dockerfile

A recipe for building a container image.

## Container Image

A runnable artifact that packages an app and its runtime requirements.

## Docker Compose

A local orchestration file for running multiple services together.

## Environment Variable

A runtime value that can change per environment without changing source code.

## Secret

A sensitive value such as an API key, password, private token, or signing key.

## Quality Gate

An automated rule that must pass before a change moves forward.

## Health Check

An endpoint or command that reports whether a service is alive.

## Logs

Events emitted by the application so humans and tools can understand what happened.

## Metrics

Numeric measurements of system behavior such as latency, errors, and resource usage.

## Runbook

An operational guide for repeating a process or responding to a known situation.

## Rollback

Returning to a previous stable version after a release fails.

## Infrastructure As Code

Defining infrastructure with versioned files instead of manual clicks.
