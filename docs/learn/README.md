# Learn DevOps With ReDevOps Lab

This folder explains the concepts that ReDevOps Lab surfaces in reports. It is written for someone who is starting DevOps and needs a practical path, not a glossary dump.

## Suggested Order

1. `00-devops-basics.md`
2. `01-runtime-configuration.md`
3. `02-containers.md`
4. `03-ci-cd.md`
5. `04-security.md`
6. `05-observability.md`
7. `06-deployment-and-iac.md`
8. `glossary.md`

## How To Use These Notes

- Read the concept before doing the related lab.
- Make one small change in a branch.
- Validate the change with the command or checklist from the report.
- Re-run ReDevOps Lab and compare the new evidence.

The goal is not to learn every tool at once. The goal is to build a habit: evidence, practice, validation, feedback.
